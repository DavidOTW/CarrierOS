# CarrierOS v0.16 Test Plan

## Principles

- Tests are deterministic and call no production API.
- Financial expected values are fixtures, not generated by the code under test.
- Tenant tests use at least two organizations and attempt both read and mutation paths.
- Migrations run on copies and compare records and money totals.
- Public-copy tests fail when unsupported features are presented as available.
- Mobile, provider-failure, and full workflow tests arrive with the phase that implements those capabilities.

## Financial parity method

1. Freeze expected outputs from the protected legacy version before editing formulas.
2. Cover profit split, contractor gross split, owner-operator split, flat per load, loaded mile, total mile, and day rate.
3. Run the legacy and Decimal implementations from the same fixture inputs.
4. Compare each stored currency result at two decimals and decisions exactly.
5. Record any difference as an explicit reviewed fixture change; never update expected values merely to make a test pass.
6. Before Decimal activation, refresh/add fixtures directly from the reviewed OTW workbook and five beta-carrier records.

## Required end-to-end acceptance test

The master 42-step scenario remains a release-level test. It must prove manual pre-booking, three driver/equipment comparisons, exactly-once booking, scanned multi-page RateCon review with three flagged conflicts, approved assignment and dispatch, mobile driver status/documents, detention, invoice packet, partial/final payment, approved/acknowledged settlement, actual-profit variance, close, no field re-entry, and complete denial to a second organization.

PR 1 does not pretend this test passes before its dependent features exist.

## Reporting

Every PR records exact passed/failed/skipped test counts, dependency-scan result, static-analysis result, migration result, coverage when configured, known warnings, and deferred cases. A passing test suite never overrides the human review gates for financial, authentication, billing, migration, or tenant-isolation changes.
